## Build Commands

```bash
make                # Build all pages and blogs (requires local php)
make docker-build   # Build via Docker (no local PHP needed)
make clean          # Remove all generated HTML files
make docker-clean   # Clean via Docker
```

The build compiles `.php` templates to `.html` files and runs `update_service_worker.sh` to update the cache manifest in `sw.js` with MD5 checksums of all generated files.

> **Note:** `vendor/parsedown` is a git submodule. Run `git submodule update --init` after cloning before building.

## Architecture

This is a PHP-based static site generator for a personal portfolio/blog. There is no npm, no bundler, and no test suite — just `make` + `php`.

### Build Pipeline

- `src/pages/*.php` → compiled via `php` → `*.html` at repo root
- `src/content/*.md` → processed through `src/content/blog.php` → `blogs/*.html`
- After HTML generation, `update_service_worker.sh` rewrites `sw.js` with current file checksums for PWA cache invalidation

### Blog Post Format

Each `.md` file in `src/content/` uses YAML frontmatter between `---` delimiters, followed by the Markdown body:

```yaml
---
title: Post Title
date: YYYY-MM-DD
tags:
  - tag1
  - tag2
description: Short blurb shown on the listing page.
external_url: https://dev.to/...   # omit for local posts
external_site: dev.to              # omit for local posts
type: video                        # or "podcast"; omit for articles
---

Markdown content here...
```

`src/utils/blog.php` handles parsing this format (using the vendored Spyc library at `vendor/spyc/`) and converts Markdown to HTML using the vendored Parsedown library (`vendor/parsedown/`).

- `external_url` being set marks the post as external (card links out, badge shown)
- `external_url` absent means the post renders locally at `/blogs/<filename>.html`
- `description` is rendered as Markdown and used as the blurb on the listing page

### Key Files

| File | Role |
|------|------|
| `src/pages/index.php` | Main landing page template |
| `src/content/blog.php` | Blog post template (renders individual blog pages) |
| `src/utils/blog.php` | Blog parsing utilities (metadata extraction, Markdown conversion) |
| `src/fragments/header.php` | Shared HTML head, navbar, GA, service worker registration |
| `src/fragments/footer.php` | Shared footer |
| `sw.js` | Service worker — **do not edit manually**, regenerated by `update_service_worker.sh` |
